# Memory-Based Model Editing at Scale

## 一句话总结
这篇论文认为，把编辑直接写进参数不够灵活，于是改成把编辑存进外部记忆，再让模型按需读取。

## 论文做了什么
作者指出，很多参数编辑方法难以精确控制编辑的适用范围，容易把本来只该影响一小块输入的改动扩散到更大范围，而且做多次编辑后性能还会继续掉。SERAC 的目标是用更高容量、更可控的方式管理编辑。

## 核心方法
SERAC 采用 semi-parametric 思路，把编辑显式存进外部 memory，再用 retrieval-augmented counterfactual model 判断当前输入是否应该调用这些编辑。换句话说，它不执着于把改动硬写进模型本体，而是让模型学会在需要时参考外部修正。

## 结果与意义
在问答、事实核验和对话生成等编辑任务上，SERAC 持续优于当时其他方法。它的重要意义在于把知识编辑从纯参数修改扩展到显式记忆检索，让编辑的作用范围和可扩展性更容易控制。
