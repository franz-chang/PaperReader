# Recommender Systems with Generative Retrieval

## 一句话总结
这篇论文把推荐系统里的检索改写成生成物品标识的过程，用生成模型直接产出下一个候选物品。

## 论文做了什么
传统推荐检索通常是把用户和物品映射到同一个向量空间，再做近邻搜索。作者认为这套范式并不是唯一选择，于是提出生成式检索思路：不给模型一个向量去找最近邻，而是让模型自回归地生成目标物品的语义 ID。

## 核心方法
论文先为每个物品构造由多个 codeword 组成的 Semantic ID，再把用户历史交互序列转换成 ID 序列，用一个 Transformer 式的 seq2seq 模型来预测下一个物品的 Semantic ID。这样一来，推荐问题被改写成了序列生成问题，而不是传统的向量匹配问题。

## 结果与意义
实验结果显示，这种生成式检索在多个数据集上优于当时的主流方法，而且对冷启动或低频物品更友好，因为 Semantic ID 带来了更强的语义泛化能力。这篇论文基本奠定了后续生成式推荐里语义 ID 加自回归生成的主线。
